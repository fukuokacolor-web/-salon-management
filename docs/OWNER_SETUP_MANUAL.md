## 1. はじめに

### 🎉 このシステムで何ができるの？

このシステムを導入すると、あなたのサロンで以下のことが **自動で** できるようになります。

- 📱 **LINEからお客様が24時間いつでも予約できる**（カレンダーをタップするだけ）
- 💾 **お客様情報・来店履歴・ポイントが自動で蓄積される**
- 📊 **ダッシュボードで本日の予約・売上・コース残回数が一目でわかる**
- 💌 **前日に自動でリマインドLINEが届く**（ドタキャン防止）
- 🔐 **お客様ご自身がポイント残高や予約を確認できる**
- 🗄 **毎週日曜に自動でバックアップが取られる**

### 📋 準備するもの一覧（全部無料でOK）

| # | 必要なもの | 補足 |
|---|---|---|
| 1 | Googleアカウント | Gmail を使っている方ならOK |
| 2 | LINE公式アカウント | 無料プランでOK（月200通まで無料）|
| 3 | PC（WindowsでもMacでも可）| スマホだけでは設定できません |
| 4 | 開発者から送られる **引き渡しパック** | ①スプレッドシートの共有リンク ②GASプロジェクトの共有リンク ③本マニュアル ④**管理画面URL**（パターンAの場合）⑤**運用形態**（パターンA / パターンBのどちらか。Step 8参照） |

> ⚠️ **開発者から「引き渡しパック」がまだ届いていない場合** は、先に開発者に連絡してください（最終章「困ったら連絡先」参照）。

## Step 1: Googleスプレッドシートをコピーする

⏱ **所要時間の目安: 約10分**

### 🤔 なぜやるの？
このシステムは、**Googleスプレッドシート（Google版のエクセルのこと）** をデータベース（お客様の情報をしまう箱）として使います。開発者から共有されたお手本のスプレッドシートを **自分用にコピー** することで、あなた専用のデータ箱が手に入ります。

### 🛠 操作手順

1. 開発者から届いたメールの **「スプレッドシート共有リンク」** をクリックします。
2. Googleスプレッドシートが開きます。
3. 画面左上の **「ファイル」** メニューをクリック → **「コピーを作成」** を選びます。
4. 「名前」欄に `〇〇サロン_顧客管理` のように分かりやすい名前を入れます。
5. 「フォルダ」は「マイドライブ」のままでOK。右下の **「コピーを作成」** ボタンをクリック。
6. コピーされた自分用スプレッドシートが新しいタブで開きます。

### 📝 このスプレッドシートIDをメモしておく

ブラウザのアドレスバー（URLが表示されている場所）を見てください。こんなURLになっています。

```
https://docs.google.com/spreadsheets/d/【ここの長い文字列がID】/edit
```

`/d/` と `/edit` の間の **長い文字列（40文字くらい）** を、メモ帳にコピーして保存しておきます。あとで Step 3 で使います。

### ✅ 確認すること
- タブのタイトルが「〇〇サロン_顧客管理」になっている
- シート下部に `予約台帳`、`顧客マスター`、`商品マスター` などのタブが並んでいる

### ⚠️ うまくいかない場合
- 「権限がありません」と表示される → 開発者に「閲覧権限をください」と連絡する
- コピーが見当たらない → Google ドライブ（https://drive.google.com）を開いて検索する

### ☑ このステップで完了したこと
- [ ] 自分用のスプレッドシートができた
- [ ] スプレッドシートIDをメモした

## Step 2: LINE公式アカウントを作る

⏱ **所要時間の目安: 約30分**

### 🤔 なぜやるの？
お客様がLINEから予約できるようにするには、**サロン専用のLINEアカウント（LINE公式アカウント）** が必要です。無料プランでOKです。

### 🛠 操作手順（A）LINE公式アカウントを作る

1. ブラウザで https://www.linebiz.com/jp/entry/ を開きます。
2. **「LINE公式アカウントをはじめる」** の青いボタンをクリック。
3. **「無料でアカウント開設」** を選び、案内に従って進めます。
4. サロン名・業種（美容）・連絡先メールを入力 → アカウント作成完了。

### 🛠 操作手順（B）Messaging API を有効にする

お客様との自動やりとり（予約ボット）を動かすには、**Messaging API（メッセージを自動で送受信する仕組み）** を有効にします。

1. LINE公式アカウントの管理画面（LINE Official Account Manager）を開きます。
2. 画面右上の **「設定」**（歯車アイコン）をクリック。
3. 左メニューの **「Messaging API」** をクリック。
4. **「Messaging APIを利用する」** ボタンをクリック。
5. LINE Developers というサイトに飛びます。
   - 💡 **ログイン方法**: 普段お使いの **個人のLINEアカウントでログインできます**。LINEアプリに登録しているメールアドレス＋パスワード、または画面に出るQRコードをスマホのLINEカメラで読み取る方法でもログインできます。
6. ここで「プロバイダー」を選ぶ画面が出ます。新規なら「新規プロバイダー作成」→ サロン名を入力。

> 📚 **「プロバイダー」と「チャネル」って何が違うの？**
>
> - **プロバイダー** ＝ 会社・組織の「入れ物」。例: 「〇〇サロン株式会社」という大きな箱。
> - **チャネル** ＝ そのプロバイダーの中にある **LINE公式アカウント1つ分の設定** のこと。
> - つまり「プロバイダー（会社）の中に、チャネル（LINEアカウント）が入っている」という **親子関係** です。サロンが1つなら、プロバイダー1つ + チャネル1つでOK。

### 🛠 操作手順（C）チャネルアクセストークン・チャネルシークレットを取得

この2つの **合言葉（トークン・シークレット）** が、システムとLINEをつなぐカギになります。

1. LINE Developers の画面で、作ったチャネル（=サロンのLINEアカウント）をクリック。
2. タブが「チャネル基本設定」「Messaging API設定」などに分かれています。

**【チャネルシークレットの取得】**

3. 上部タブの **「チャネル基本設定」** をクリック。
4. 下にスクロールすると **「チャネルシークレット」** という項目があり、長い英数字が表示されています。
5. 右の **「コピー」ボタン** をクリックしてメモ帳に貼り付け、`LINE_CHANNEL_SECRET` とラベルをつけて保存。

**【チャネルアクセストークンの取得】**

6. 上部タブの **「Messaging API設定」** をクリック。
7. 一番下までスクロールすると **「チャネルアクセストークン（長期）」** 欄があります。
8. 最初は空欄なので、**「発行」ボタン** をクリック。長い文字列が生成されます。
9. コピーしてメモ帳に貼り付け、`LINE_CHANNEL_ACCESS_TOKEN` とラベルをつけて保存。

### ⚠️ うまくいかない場合
- 「Messaging APIを利用する」ボタンが見当たらない → LINE公式アカウント管理画面の「設定」→「アカウント設定」から試す
- トークン発行ボタンを押しても何も起きない → ブラウザを再起動・別タブで開き直す

### ☑ このステップで完了したこと
- [ ] LINE公式アカウントが作れた
- [ ] Messaging API が有効になった
- [ ] チャネルアクセストークン（長い文字列）をメモした
- [ ] チャネルシークレット（長い文字列）をメモした

## Step 3: GAS（プログラム）を自分用に設定する

⏱ **所要時間の目安: 約30分**

### 🤔 なぜやるの？
**GAS（ジーエーエス、Google Apps Script の略。Googleが提供する無料のプログラム実行環境のこと）** が、このシステムの本体です。開発者から共有されたGASプロジェクトを、あなたのGoogleアカウントで動くように **自分用に複製 & 設定** します。

### 🛠 操作手順（A）GASプロジェクトを自分用にコピー

1. 開発者から届いたメールの **「GASプロジェクト共有リンク」** をクリック。
2. GASエディタ（プログラムが書かれている画面）が開きます。
3. 画面左上の **「ファイル」** → **「コピーを作成」** を選びます。
   - ※「オーバーフロー」の3点メニュー内にある場合もあります
4. コピーが作成され、自分専用のプロジェクトが開きます。名前を `〇〇サロン_GAS` に変えておきます。

> 💡 **代替手段**: 開発者が事前に Step 1 でコピーしたスプレッドシートに紐づくGASを用意してくれている場合は、このコピー作業は不要です。スプレッドシートの **「拡張機能」→「Apps Script」** で開くだけでOK。どちらか分からない場合は開発者に確認してください。

### 🛠 操作手順（B）スクリプトプロパティを設定する

**スクリプトプロパティ（プログラムに覚えさせる「合言葉」の保管庫のこと）** に、LINEのトークンなどを登録します。

GASエディタで、左メニューの **「⚙ プロジェクトの設定」**（歯車アイコン）をクリック。下までスクロールすると **「スクリプト プロパティ」** の項目があり、**「スクリプトプロパティを追加」** ボタンで1つずつ追加できます。

#### 🔑 必須プロパティ（必ず設定するもの）

| プロパティ名（キー） | 入れる値 | 意味 |
|---|---|---|
| `LINE_CHANNEL_ACCESS_TOKEN` | Step 2 でメモした長期トークン | LINEと通信するための合言葉 |
| `LINE_CHANNEL_SECRET` | Step 2 でメモしたシークレット | LINEの本物確認用 |
| `SPREADSHEET_ID` | Step 1 でメモしたID | データ箱の場所 |
| `STORE_EMAIL` | サロンのメールアドレス | エラー通知の受け取り先 |
| `ADMIN_PASSWORD` | 自分で決める任意のパスワード（8文字以上推奨） | 管理画面にログインする時のパスワード |

#### ⏳ 後から設定するもの（今は空でOK）

| プロパティ名（キー） | いつ設定するか |
|---|---|
| `OWNER_LINE_ID` | Step 9 のテスト時に「マイID」とボットに送ると自動表示されるので、それをメモして後で登録 |
| `ADMIN_PASSWORD_HASH` | **自動で作られます**（Step 8 で初めて管理画面にログインしたとき、入力された `ADMIN_PASSWORD`（平文）がGAS側に送られ、**SHA-256で安全な形にハッシュ化されて** この `ADMIN_PASSWORD_HASH` に保存されます。同時に **平文の `ADMIN_PASSWORD` はスクリプトプロパティから自動的に削除** されます）|
| `BACKUP_FOLDER_ID` | **自動で作られます**（Step 10 の `weeklyBackup` **初回実行時** にDrive内にフォルダが自動作成されます。`installTriggers` を実行しただけではフォルダは作られず、毎週日曜2時の初回自動実行まで待つことになります。すぐ作りたい場合は関数ドロップダウンから `weeklyBackup` を手動実行してください）|
| `RICH_MENU_ID` | **自動で作られます**（Step 7 の `setupRichMenu` 実行時）|
| `WEB_API_KEY` | **自動で作られます**（管理画面から最初にAPIアクセスした際にGAS側で自動生成・保存される管理画面APIキー）|

#### 🎨 任意プロパティ（必要なら）

| プロパティ名 | 内容 |
|---|---|
| `MENU_LIST` | メニュー一覧（カンマ区切り）例: `カット,カラー,トリートメント`。未設定時は開発者デフォルト |
| `SALON_IMAGE_URL` | LINE Flexカード用のサロン画像URL |
| `RICHMENU_IMAGE_URL` | リッチメニュー背景画像URL |
| `STRIPE_SECRET_KEY` | Stripe（クレカ決済）の秘密キー。**β版では使いません** |
| `STRIPE_WEBHOOK_SECRET` | StripeのWebhook署名検証用シークレット。**β版では使いません** |
| `EXCHANGE_URL` | ポイント交換用EC画面のURL。**β版では使いません** |

> 🛈 **β版運用時の注意**: 上記表の `STRIPE_SECRET_KEY` / `STRIPE_WEBHOOK_SECRET` / `EXCHANGE_URL` は **EC機能（Stripe決済・ポイント交換EC）用** です。β版ではEC機能は使わないため、**これらは設定不要** です。正式運用で決済機能を有効化する際に開発者と一緒に設定します。

### ⚠️ うまくいかない場合
- 「スクリプトプロパティ」が見当たらない → 左メニュー一番下の **歯車⚙** をクリック。中段〜下の方にあります
- 値を入れて「保存」したのに消えてしまう → もう一度「保存」ボタンを押してください。ブラウザ再読み込みで確認

### ☑ このステップで完了したこと
- [ ] GASプロジェクトが自分のGoogleアカウントに入った
- [ ] 必須5プロパティを全部登録した
- [ ] 自分の `ADMIN_PASSWORD` を忘れないようメモした

> 💡 **順序に関する豆知識**: このマニュアルは基本的に **Step 3 → Step 4（デプロイ）→ Step 6（initializeSpreadsheet 実行）** の順で進めます。ただし `initializeSpreadsheet` は Step 3 直後（Step 4 のデプロイ前）に実行しても問題なく動きます。その順序にすると、GASの **承認画面が1回で済む** ので少し楽です。基本順で進めて構いませんが、どちらでもOKということだけ覚えておいてください。

## Step 4: ウェブアプリとして公開する（デプロイ）

⏱ **所要時間の目安: 約10分**

### 🤔 なぜやるの？
**デプロイ（公開設定のこと）** を行うと、GASプログラムに **専用のURL** が発行されます。このURLがあって初めて、LINEからこのシステムに予約データを渡せるようになります。

### 🛠 操作手順

1. GASエディタの **画面右上の青い「デプロイ」ボタン** をクリック。
2. メニューが出るので **「新しいデプロイ」** を選びます。
3. 左上の **歯車アイコン⚙** → **「ウェブアプリ」** を選択。
4. 設定画面が開くので、以下のとおり設定：
   - **「説明」**: `初回デプロイ` など自由
   - **「次のユーザーとして実行」**: `自分（あなたのメールアドレス）`
   - **「アクセスできるユーザー」**: **「全員」** を選択（重要！LINE からアクセスするため）
5. 右下の青い **「デプロイ」** ボタンをクリック。
6. 「承認が必要です」と出たら **「アクセスを承認」** → Googleアカウント選択 → 「詳細」→「〇〇（安全ではないページ）に移動」→ 「許可」。
   > ⚠️ 「安全ではない」と出ますがこれは自分のスクリプトなので問題ありません。
7. 完了画面に **「ウェブアプリURL」** が表示されます。これをコピーしてメモ帳に保存！

> 💡 **初回と2回目以降のデプロイの違い（重要）**
>
> - **初回**: 上記のとおり **「新しいデプロイ」** を選びます。新しいURLが発行されます。
> - **2回目以降（コード修正時）**: 「新しいデプロイ」を使うと **URLが変わってしまい**、LINEのWebhook URLも毎回貼り直しになります。コード修正を反映させたいだけのときは、デプロイメニューの **「デプロイを管理」** を開き、既存デプロイの右側にある **鉛筆（編集）アイコン** をクリック → バージョンを「新バージョン」に切り替えて **「デプロイ」** をクリック。これで **URLを変えずにコードだけ更新** できます。

### 📝 URLの形は？

```
https://script.google.com/macros/s/【長いID】/exec
```

この **【長いID】** を **デプロイID** と呼びます。Step 5 でLINEに登録します。

### ⚠️ うまくいかない場合
- 「承認」画面が繰り返し出る → ブラウザのポップアップブロックを解除
- デプロイボタンがグレーアウト → 左のファイル一覧からどれか1つクリックしてから再挑戦

### ☑ このステップで完了したこと
- [ ] ウェブアプリURLがメモできた
- [ ] アクセスできるユーザーを「全員」にした

## Step 5: LINEのWebhook設定

⏱ **所要時間の目安: 約10分**

### 🤔 なぜやるの？
**Webhook（ウェブフック、LINEにメッセージが届いたらこのURLに通知して！と伝える仕組み）** を設定すると、お客様がLINEで送ったメッセージをシステムが受け取れるようになります。

### 🛠 操作手順

1. LINE Developers のサイトを再度開き、あなたのチャネルの **「Messaging API設定」** タブを開きます。
2. **「Webhook設定」** の **「Webhook URL」** 欄に、Step 4 でコピーしたURL（`https://script.google.com/macros/s/xxx/exec`）を貼り付けます。
3. 右の **「更新」** ボタンをクリック。
4. **「検証」** ボタンをクリック → 「成功」と出ればOK！
5. **「Webhookの利用」** のトグルを **ON** にします。

### ⚠️ 重要: 自動応答機能をOFFにする

LINE公式アカウント管理画面に戻って以下の設定を変更します。これをやらないと、LINEの自動応答がシステムと **ケンカして** 予約が動かなくなります。

1. LINE Official Account Manager（管理画面）を開く。
2. **「設定」（歯車）→「応答設定」**。
3. **「応答メッセージ」→ オフ**
4. **「あいさつメッセージ」→ オフ**（任意だが推奨）
5. **「Webhook」→ オン**

### ⚠️ うまくいかない場合

## Step 8: 管理画面にログインしてみる

⏱ **所要時間の目安: 約10分**

### 🤔 なぜやるの？
ブラウザから予約一覧・顧客一覧・売上を見るための **管理画面（ウェブサイト）** にログインできるか確認します。

### 📌 重要: 管理画面URLの運用について

管理画面は2つの運用パターンがあります。どちらになっているか開発者に確認してください。

- **パターンA（共有型）**: 開発者が1つのURLを運用し、`js/config.js` の `GAS_URL` だけ各サロンごとに差し替える形（最も手軽）
- **パターンB（複製型）**: サロンごとにGitHubをfork（コピー）して、各サロン専用のURLを持つ形

> 💬 **どちらなのか分からない場合は、開発者に「管理画面の運用形態はA・Bどちらですか？」と聞いてください。**

### 🛠 操作手順

1. 開発者から教わった **管理画面URL** をブラウザで開きます（例: `https://xxxx.github.io/salon-repo/`）。
2. ログイン画面が出ます。
3. **パスワード欄** に、Step 3 で設定した `ADMIN_PASSWORD` の値を入力。
4. **「ログイン状態を保持する」** チェックは、自分専用PCならON、共有PCならOFFを推奨。
5. **「ログイン」** ボタンをクリック → ダッシュボードが表示されれば成功！

### 🎨 自分好みの文言を入れる（運用設定タブ）

1. 左メニューから **「設定」** をクリック。
2. **「運用設定」** タブを開きます。
3. **キャンセルポリシー文言** を自分のサロン用に書き換える。例:
   ```
   ※ キャンセルは前日までにお願いします。当日キャンセルは50%、無断キャンセルは100%のキャンセル料がかかります。
   ```
4. **「保存」** ボタンをクリック。

### ⚠️ うまくいかない場合
- 「パスワードが違います」→ Step 3 でメモしたパスワードを確認
- 画面が真っ白 → ブラウザを再読み込み（F5キー）、`js/config.js` の `GAS_URL` が正しいか開発者に確認

### ☑ このステップで完了したこと
- [ ] 管理画面にログインできた
- [ ] キャンセルポリシーを自分のサロン仕様に書き換えた

## Step 9: テスト予約をしてみる

⏱ **所要時間の目安: 約15分**

### 🤔 なぜやるの？
ここまでの設定がすべて正しく動くか、 **自分のスマホで実際に予約して確認** します。

### 🛠 操作手順

1. スマホのLINEアプリを開きます。
2. LINE Official Account Manager の「友だちを増やす」→「友だち追加ガイド」からQRコードまたはIDを取得。

4. トーク画面を開き、まず **「マイID」** と送信します。
5. `U` から始まる長い文字列（あなたのLINE ID）が返ってきます。→ **これをコピーしてメモ**。
6. GASエディタに戻って、スクリプトプロパティの `OWNER_LINE_ID` にこの値を登録（Step 3 参照）。

### ✅ 確認すること

以下を上から順にチェックしてください。どれか1つ外れていると、ボットから返事が来ません。

- [ ] **①Webhook利用がON？** — LINE Developers の該当チャネル → Messaging API設定 → 「Webhookの利用」トグルがONになっているか

- [ ] **④友だち追加できている？** — サロン公式アカウントが「友だち一覧」に本当に入っているか。ブロック状態になっていないか

## こんなとき どうする（FAQ）

### ❓ LINEからの予約が届かない

以下を上から順に確認してください。

- [ ] Step 5 の **Webhook URL** が正しく貼られているか？
- [ ] Webhook検証で「成功」になっているか？
- [ ] **Webhookの利用** がONになっているか？
- [ ] **応答メッセージ** がOFFになっているか？
- [ ] LINE Developers の **Messaging API設定 → 応答メッセージ** もOFFか？

### ❓ 管理画面にログインできない（パスワードを忘れた）

1. GASエディタを開く。
2. 左メニュー **⚙ プロジェクトの設定 → スクリプトプロパティ**。
3. **`ADMIN_PASSWORD_HASH`** の行を **削除**。
4. **`ADMIN_PASSWORD`** に新しいパスワードを入れる（または既存値を書き換える）。
5. 次回管理画面ログイン時、入力したパスワードで再度ハッシュ化が自動で行われます。

### ❓ バックアップが警告色（オレンジ）になっている

> 💡 **前提**: `BACKUP_FOLDER_ID` は `weeklyBackup` を **初めて実行したとき** にDrive内にフォルダが作られて自動保存されます。`installTriggers`（Step 10）を実行しただけではフォルダは作られず、毎週日曜2時の初回自動実行まで何もできません。セットアップ直後にバックアップが警告色で表示されるのは正常ですが、すぐ緑にしたい場合は以下で手動実行してください。

1. GASエディタを開く。
2. 関数ドロップダウンから **`weeklyBackup`** を選択。
3. **「▶ 実行」** をクリック。
4. 初回は承認画面が出るので許可。完了後、Driveに「サロン顧客管理_バックアップ」フォルダが自動作成される。
5. 管理画面ダッシュボードを再読み込みし、緑になっていればOK。

### ❓ スプレッドシートの列を間違えて消した／おかしくなった

1. Google ドライブ（https://drive.google.com）を開く。
2. 左メニューから **「マイドライブ」→「サロン顧客管理_バックアップ」** フォルダを探す。
3. 最新日付の `サロン顧客管理_バックアップ_yyyyMMdd_HHmm` ファイルを右クリック → **「コピーを作成」**。
4. コピーを開いて必要な行をコピー → 元のスプレッドシートに貼り付け。
5. 万一全部ダメな時は、そのバックアップを新しい本番として使う（その場合、スクリプトプロパティの `SPREADSHEET_ID` を更新）。

### ❓ 顧客が「パスワードを忘れた」と言ってきた

1. 管理画面にログイン。
2. **「顧客一覧」** からその方の名前をクリック。
3. **「🔑 ログインパスワード」** カードの **「パスワードをリセット」** オレンジボタンをクリック。
4. 確認ダイアログで「OK」→ リセット完了。
5. お客様にLINEで「パスワードをリセットしました。もう一度ご登録ください」と連絡。

### ❓ 予約のキャンセル通知がオーナーに届かない

- スクリプトプロパティ `OWNER_LINE_ID` が正しく設定されているか確認（Step 9 参照）。

### ❓ 新しいスタッフを追加したい

- このシステムは現在 **オーナー1人＋お客様** の構成です。スタッフ個別ログインは未実装。必要な場合は開発者に相談を。

### ❓ Webhook検証で「401」や「403」エラーが出る

Step 5 の「検証」ボタンを押したら `401 Unauthorized` / `403 Forbidden` と表示される場合、ほぼ間違いなく **デプロイ設定の問題** です。

1. GASエディタ右上の **「デプロイ」→「デプロイを管理」** を開く。
2. 現在のデプロイの **鉛筆アイコン** をクリック。
3. **「アクセスできるユーザー」** が **「全員」** になっているか確認（「自分のみ」「ドメイン内のユーザー」だとLINEからアクセスできず401/403になります）。
4. 「全員」に変更 → 「デプロイ」でURLを保ったまま更新。
5. LINE Developers の Webhook URL は同じまま「検証」を再実行。

### ❓ スプレッドシートIDを間違えて `SPREADSHEET_ID` に設定してしまった

「別のスプレッドシートにデータが書き込まれてしまう」「シートが見つからないエラー」が出るとき。

1. 正しいスプレッドシートを開き、アドレスバーから `/d/` と `/edit` の間の **長い文字列（40文字程度）** をコピー（Step 1 参照）。
2. GASエディタ → ⚙ プロジェクトの設定 → スクリプトプロパティ。
3. `SPREADSHEET_ID` の行の値を **正しいIDに書き換えて保存**。
4. もし間違ったスプレッドシートに予約が書き込まれていた場合は、そのデータを手動でコピーして正しいスプレッドシートに貼り付け。
5. 念のため `initializeSpreadsheet` を再実行し、必要なシートが揃っているか確認。

### ❓ LINEの月200通無料枠を超えそう／超えた

LINE公式アカウントの **コミュニケーションプラン** は月200通まで無料。超過するとお客様に配信が届かなくなります。

- **状況確認**: LINE Official Account Manager → 「分析」→「メッセージ通数」で今月の送信数を確認。
- **対策1（節約）**: リマインドLINEの内容を簡潔にする／不要な一斉送信を減らす。
- **対策2（プラン変更）**: LINE Official Account Manager → 「設定」→「利用と請求」→「プラン変更」から **ライトプラン（月5,000円で5,000通）** や **スタンダードプラン** にアップグレード。
- **対策3**: システム側でリマインド送信をOFFにしたい場合は、Step 10 の `installTriggers` で登録された「dailyReminder」トリガーをGASエディタの⏰トリガー画面から削除。
